# Sony PaSoRi RC-S300 PC/SC 指令

本文记录 RC-S300 的 PC/SC 状态模型、APDU 指令和 FeliCa 通信方式。设备在系统中通常
显示为 `SONY FeliCa Port/PaSoRi 4.0`。

## 通信入口

- 普通卡片 APDU 在卡片捕捉后通过 `SCardTransmit` 发送。
- Transparent Mode 必须通过 `SCardControl` 发送，控制码为 `SCARD_CTL_CODE(3500)`。
- CCID 类透明命令使用 `FF C2`，设备专用类透明命令使用 `FF 50`。
- Transparent Mode 与普通卡片操作互斥。进入后设备不再检测卡片放置或移除。

## APDU 一览

| 指令 | APDU | 用途 |
|------|------|------|
| Get Data | `FF CA` | 获取卡片 ID、类型、名称和通信速度 |
| Load Keys | `FF 82` | 注册 MIFARE Classic 密钥 |
| General Authenticate | `FF 86` | MIFARE Classic 块认证 |
| Read Binary | `FF B0` | 读取卡片数据 |
| Update Binary | `FF D6` | 写入卡片数据 |
| Increment / Decrement | `FF C2 00 03` | MIFARE value block 操作 |
| Select File | `FF A4` | 选择 FeliCa 服务或 MIFARE Ultralight sector |
| Data Exchange | `FF FE` | FeliCa 原生通信 |
| Manage Session | `FF C2/50 00 00` | 管理 Transparent Session 和 RF |
| Transparent Exchange | `FF C2/50 00 01` | 透明收发 |
| Switch Protocol | `FF C2/50 00 02` | 选择 RF 技术和激活层级 |
| Reset Device | `FF 55` | 复位设备 |
| Get Firmware Version | `FF 56` | 获取固件和 bootloader 版本 |
| Diagnose | `FF 57` | 设备自检 |
| Set RF Speed | `FF 5C` | 设置各 RF 技术的速度上限 |
| Get RF Speed | `FF 5D` | 获取各 RF 技术的速度上限 |
| Get Property | `FF 5F` | 获取序列号和机型信息 |

ISO/IEC 14443-4 卡还可以直接发送卡片支持的任意 APDU。FeliCa Dual 卡的普通 APDU
由设备转换为 FeliCa 通信。

## 设备状态

设备有三种状态：

1. 探卡：没有捕捉到卡片。
2. 捕捉：卡片已被 PC/SC 捕捉。
3. Transparent：透明会话已开始。

| 指令 | 探卡 | 捕捉 | Transparent |
|------|------|------|-------------|
| Get Data | - | Y | Y |
| Load Keys | Y | Y | Y |
| General Authenticate | - | Y | Y |
| Read/Update Binary | - | Y | - |
| Increment/Decrement | - | Y | - |
| Select File | - | Y | - |
| Data Exchange | - | Y | - |
| Manage/Exchange/Switch | - | - | Y |
| Firmware/Diagnose | Y | Y | Y |
| Set/Get RF Speed | Y | Y | - |

Start Transparent Session 是进入 Transparent 状态的入口。会话开始时 RF 会关闭，已有
卡片连接会解除。

## APDU 长度

- 指令需要 Data In 时必须带 `Lc`，且 `Lc` 必须等于 Data In 长度。
- 指令不需要 Data In 时不要带 `Lc`。
- `Le=00` 表示读取最大长度。
- `Le` 小于实际响应长度时返回 `6C XX`，`XX` 是实际长度。
- `Le` 大于实际响应长度时返回 `62 82`。

## Get Data

```text
FF CA <P1> 00 <Le>
```

| P1 | 返回内容 |
|----|----------|
| `00` | 卡片 ID；FeliCa 返回 8 字节 IDm |
| `01` | 捕捉信息；FeliCa 返回 8 字节 PMm |
| `F0` | 卡片识别 ID |
| `F1` | NULL 结尾的卡片名称 |
| `F2` | 当前通信速度：设备到卡、卡到设备各一字节 |
| `F3` | 卡片技术类型 |
| `F4` | NULL 结尾的卡片技术名称 |

通信速度值为 `01=106`、`02=212`、`03=424`、`04=848 kbps`。卡片技术类型为
`01=Type A`、`02=Type B`、`04=FeliCa`、`07=Type A T=CL`、`08=Type B T=CL`、
`0B=Type V`。

## MIFARE 认证

### Load Keys

```text
FF 82 00 <key-number> 06 <key:6>
```

`key-number` 为 `00` 或 `01`。密钥在设备断电、设备复位或 USB reset 后清除。

### General Authenticate

```text
FF 86 00 00 05 00 <block-msb> <block-lsb> <key-type> <key-number>
```

`key-type` 为 `60`（Key A）或 `61`（Key B）。

## FeliCa 结构化读写

结构化读写先通过 Select File 设置服务列表，再使用 Read Binary 或 Update Binary。服务
和区域最多各 16 个，只支持无安全认证的服务。

### Select File

```text
FF A4 <area-count> <service-count> <Lc> <area-codes> <service-codes>
```

节点码为 2 字节，`Lc = (area-count + service-count) * 2`。仅选择无安全认证服务时不需要
区域码。

### Read Binary

```text
FF B0 <address-msb> <address-lsb> [Lc <block-list>] <Le>
```

FeliCa address 有两种形式：

- `b15=0`：`b14..12` access mode，`b11..8` service index，`b7..0` block index。
- `b15=1`：`b7..0` 为 block 数，详细 block list 放在 Data In。

每块 16 字节，单次最多读取 15 块。指定 block list 时 `Le=16 * block-count`。

### Update Binary

```text
FF D6 <address-msb> <address-lsb> <Lc> <block-list?> <block-data>
```

每块 16 字节，单次最多写入 13 块。服务列表、block list 和数据的总长度不能超过
243 字节。

## FeliCa Data Exchange

Data Exchange 只在捕捉到 FeliCa 卡时可用：

```text
FF FE <mode> <timeout> <Lc> <command> <Le>
```

| mode | 行为 |
|------|------|
| `00` THRU | 设备插入并校验捕捉卡片的 IDm；命令从 FeliCa command code 开始 |
| `01` DIRECT | 原样发送从 LEN 开始的 FeliCa frame；CRC 由设备处理 |

`timeout=00` 时根据 PMm 自动计算；`01..FE` 以 10 ms 为单位。

## Transparent Mode

Transparent Mode 用 BER-TLV data objects 控制 RF 和收发原始数据。完整流程是：

1. Start Transparent Session。
2. Switch Protocol。
3. Transparent Exchange。
4. End Transparent Session。

所有退出路径都必须执行 End Session。

### Manage Session

```text
FF C2 00 00 <Lc> <data-objects> <Le>
```

| Tag | Data object | Value |
|-----|-------------|-------|
| `80` | Version | 3 字节 RFU |
| `81` | Start Transparent Session | 空 |
| `82` | End Transparent Session | 空 |
| `83` | RF off | 空 |
| `84` | RF on | 空 |
| `5F46` | Timer | 4 字节微秒值，小端 |
| `FF6D` | Get Parameters | 嵌套 TLV |
| `FF6E` | Set Parameters | 嵌套 TLV |

开始和结束示例：

```text
FF C2 00 00 02 81 00 00
FF C2 00 00 02 82 00 00
```

### Switch Protocol

```text
FF C2 00 02 04 8F 02 <standard> <layer> 00
```

| 技术 | standard | layer |
|------|----------|-------|
| Type A | `00` | `02` 配置；`03` WUPA+SELECT；`04` 再执行 RATS+PPS |
| Type B | `01` | `02` 配置；`03` WUPB；`04` 再执行 ATTRIB |
| Type V | `02` | `02` 配置；`03` Inventory |
| FeliCa | `03` | `00` 配置；`01` Polling |

Switch Protocol 会先执行 RF off/on，再执行指定 layer。PC/SC 已捕捉 FeliCa 时可使用：

```text
FF C2 00 02 04 8F 02 03 00 00
```

### Transparent Exchange

```text
FF C2 00 01 <Lc> <data-objects> <Le>
```

| Tag | Data object | 用途 |
|-----|-------------|------|
| `90` | Transmission and Reception Flag | CRC、parity、protocol prologue 控制 |
| `91` | Transmission Bit Framing | 发送末字节有效位数 |
| `92` | Reception Bit Framing | 接收末字节有效位数 |
| `93` | Transmit | 仅发送 |
| `94` | Receive | 仅接收 |
| `95` | Transceive | 发送后接收 |
| `5F46` | Timer | 紧邻 Receive/Transceive 时表示接收超时 |

tag `90` 的默认行为是不添加或移除 CRC、不处理 protocol prologue。FeliCa 通常保持默认
framing，只提供 timer 和 tag `95`：

```text
FF C2 00 01 <Lc>
5F 46 04 <timeout-us-le:4>
95 <frame-len> <FeliCa-frame>
00
```

响应是 BER-TLV data objects 加外层 SW1/SW2：

| Tag | 内容 |
|-----|------|
| `C0 03` | data object index + 内层 SW1/SW2 |
| `92 01` | 接收末字节有效位数 |
| `96 02` | 接收状态 + RFU |
| `97` | 卡片原始响应 |

必须先检查外层状态，再检查 `C0` 和 `96`，最后从 `97` 取卡片响应。不要依赖 tag 顺序。

## 设备指令

### Get Firmware Version

```text
FF 56 00 00 <Le>
```

响应依次包含：整体版本 4 字节、MCU/SAM/RFFE/RFFE EEPROM/bootloader 版本各 2 字节、
更新状态 2 字节、启动状态 2 字节。不存在或无法获取的版本为 `FFFF`。

### Diagnose

```text
FF 57 00 00 <Lc> <test-number> <parameters> <Le>
```

| test-number | 测试 |
|-------------|------|
| `00` | 通信回环，最多 501 字节 |
| `01` | ROM CRC，返回一字节结果 |
| `02` | RAM 读写，成功返回 6 字节 `00` |
| `03` | RF Polling；参数为技术和次数，返回成功次数 |

Polling 技术值为 `00=Type A`、`01=Type B`、`02=FeliCa`、`03=Type V`。Polling test
会执行 RF off/on 并清除当前捕捉状态。

### Set/Get RF Speed

```text
FF 5C 00 00 03 <technology> <tx-speed> <rx-speed>
FF 5D 00 00 01 <technology> <Le>
```

技术值为 `00=Type A`、`01=Type B`、`02=FeliCa`；速度值为 `01=106`、`02=212`、
`03=424`、`04=848 kbps`。FeliCa 的发送和接收速度必须相同，默认上限为 424 kbps。
设置会被 Transparent Mode 继承，设备或 USB 复位后恢复默认值。

### Get Property

```text
FF 5F <P1> 00 <Le>
```

`P1=03` 返回 7 字节序列号，`P1=08` 返回 8 字节机型信息。

## 状态码

| SW | 含义 |
|----|------|
| `9000` | 成功 |
| `6282` | 响应短于 Le |
| `6401` | 卡片无响应 |
| `6700` | Lc 或命令长度错误 |
| `6800` | CLA 错误 |
| `6981` | Data In 错误 |
| `6985` | 当前设备状态不允许该指令 |
| `6A81` | INS 或功能不支持 |
| `6B00` | P1/P2 错误 |
| `6CXX` | Le 错误，XX 为实际长度或允许上限 |
| `6F00` | 设备内部错误 |

透明命令的 `C0 03 <index> <sw1> <sw2>` 指向第 `index` 个 data object。常见内层状态：

| 内层 SW | 含义 |
|---------|------|
| `6300` | 卡片或设备通信异常 |
| `6301` | data object 顺序或设备状态错误 |
| `6400` | 设备无响应 |
| `6401` | 卡片无响应 |
| `6700` | data object 长度错误 |
| `698A` | 重复开始 Transparent Session |
| `6A80` | TLV 格式错误 |
| `6A81` | 未定义 tag |

## 实现注意

- 普通卡片模式与 Transparent Mode 不得混用。
- Start Session 会解除已有卡片连接；End Session 后再恢复普通监控。
- Transparent 命令应走 `SCardControl`，不要把特定驱动接受 `SCardTransmit` 当作可移植行为。
- FeliCa Data Exchange 的自动 timeout 在无卡状态下可能为 0，应只在卡片捕捉后使用。
- Deselect、Release 或会改变 RF 状态的诊断执行后，应断开并等待 PC/SC 重新捕捉卡片。
